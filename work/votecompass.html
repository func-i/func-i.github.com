Functional Imperative

Real-time voting analysis based on an individual’s responses to a series of propositions specifically developed for a given election

Vote Compass is an interactive electoral literacy application developed by political scientists and run during election campaigns. It offers an accessible framework for learning about party apps, stimulates discussion on a wide variety of election issues, and encourages democratic participation within the electorate. Functional Imperative rebuilt Vote Compass’ infrastructure which has registered nearly 5 million full respondents to its polls and has received about 13 million unique visitors.

Minimize loading whilst maximizing up time

Functional Imperative was approached by Vote Compass as the 2013 Australian general election approached. Having partnered with the Australian Broadcasting Corporation, Vote Compass was poised to receive significant exposure and traffic within the short election cycle window. Previous experience led Vote Compass to believe that their existing infrastructure might not be able to scale for the large loads expected. On a tight deadline, Functional Imperative investigated and recommended changes to the Vote Compass infrastructure to minimize load-time, maximize up-time and ensure an optimal user experience.

Individual’s responses powered by an open database

Based on an individual’s responses to a series of propositions specifically developed for a given election, Vote Compass generates a real-time analysis of how one’s views compare to the apps of each political party vying for office. This analysis is powered by an open database which contains a public record of each party’s stand on the issues included in Vote Compass.

Working quickly, we identified issues with server response and page load times. To mitigate we:

  • Minified static assets

    Serving images, javascript, css and using a CDN network.

  • Added a cacheing layer

    Using multiple MemCache servers.

  • Added load balancers

    With automatic server scaling to handle surges in traffic.

  • Optimized long running queries

    Which were needlessly locking database rows.

  • Ran stress tests

    Ensuring performance under 2x peak load.

With the changes completed in time, Vote Compass was used by millions of visitors and handled over seven thousand requests per minute at its peak. We are now actively working with Vote Compass as they expand globally.
